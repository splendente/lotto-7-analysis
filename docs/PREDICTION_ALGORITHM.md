# 当せん番号予測アルゴリズム

本アプリケーションの「次回当せん番号予測」機能は、過去の抽せんデータをもとに各番号の傾向をスコアリングし、最も期待値の高い7つの番号を選出する独自のアルゴリズムを採用しています。

直近の抽せん結果を対象としたバックテスト（グリッドサーチ）によってハイパーパラメータ（各スコアの重み付け）を最適化し、近年のトレンドに追従するようにチューニングされています。

## 予測アルゴリズムの概要

予測は、以下の6つの指標（スコア）の合計値を「その番号の選ばれやすさ」として扱います。※カッコ内の重み付けは最新の最適化結果（直近20回特化）によるものです。

1. **待機回数スコア (Absence Score)**
   - 対象の番号が最後に当せんしてからの空き回数。
   - 長らく出ていない番号（コールドナンバー）は「そろそろ出やすい」とみなして優遇します。
   - **重み付け: `x2.0` (超重要)**

2. **連続出現スコア (Consecutive Score)**
   - 直近の抽せんで連続して当せんしている（引っ張っている）回数。
   - 勢いのある数字が次も出やすい傾向を評価します。
   - **重み付け: `x2.0` (超重要)**

3. **直近トレンドスコア (Recent Trend Score)**
   - 直近30回分の抽せんにおける出現頻度。
   - ここ最近でよく出ている「トレンドに乗っている」番号を評価します。
   - **重み付け: `x1.0` (普通)**

4. **出現頻度スコア (Overall Frequency Score)**
   - 第1回からの全データにおいて、対象の番号が当せんした回数。
   - **重み付け: `x0.0` (無視)** ※直近の傾向に合わないため意図的に除外

5. **相関性スコア (Correlation Score)**
   - 既に選出された番号リストと、過去に「同時に当せんした回数」が多い番号を評価する。
   - **重み付け: `x0.0` (無視)** ※直近の傾向に合わないため意図的に除外

6. **ボーナス昇格スコア (Bonus Promotion Score)**
   - 直近2回の抽せんで「ボーナス数字」として出現した番号が、次回「本数字」に昇格しやすいかを評価。
   - **重み付け: `x0.0` (無視)** ※直近の傾向に合わないため意図的に除外

## 抽選のフロー

1. 初期状態として、1〜37までの番号プールを用意します。
2. 以下の計算をループさせ、プールから番号が7つ選ばれるまで繰り返します。
   1. 残っているすべての番号に対して、上記の6指標をもとに**総合スコア**を計算します。
   2. 計算された各番号の総合スコアを比較し、**最もスコアが高い1つの番号**を選出（プールから除外）します。
   3. 次のループへ進みます。

## UI上の表示理由について

選出された7つの番号には、どの要素でスコアを大きく稼いで選ばれたかを示すラベルが付与されます。

- **連続出現中**: 直近の抽せんで2回以上連続で当せんしており、連続出現スコアが高い場合（最優先で表示）。
- **ボーナス昇格**: 直近でボーナス数字から本数字への昇格が期待される場合。
- **直近トレンド**: 直近30回で出現頻度が高くトレンドに乗っている場合。
- **長期待機**: 最後に当せんしてからの空き回数が相対的に長く、待機スコアが特出して高かった場合。
- **高頻度**: 全期間の出現回数が特出して高かった場合（※現在の重み設定ではほぼ出現しません）。
- **相性よし**: 既に当せん番号として選出された直前の番号たちと過去の同時出現回数が多い場合（※現在の重み設定ではほぼ出現しません）。
- **統計バランス**: 上記のどれにも極端に偏ることなく、無難なバランスで引き当てられた場合。

> 当アルゴリズムは乱数を使用せず、過去のデータに基づく完全なスコア順で選出するため、データが更新されない限り「予測する」ボタンで提案される番号は常に同じものになります。
