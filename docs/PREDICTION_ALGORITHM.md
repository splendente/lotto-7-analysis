# 当せん番号予測アルゴリズム

本アプリケーションの「次回当せん番号予測」機能は、過去の抽せんデータをもとに各番号の傾向をスコアリングし、**重み付きランダム抽選**によって7つの番号を選出する独自のアルゴリズムを採用しています。

## 予測アルゴリズムの概要

予測は、以下の3つの指標（スコア）の合計値を「その番号の選ばれやすさ」として扱います。

1. **出現頻度スコア (Frequency Score)**
   - 過去の全データにおいて、対象の番号が当せんした回数。
   - よく出る番号ほど少し選ばれやすくする。重み付け: `x1.0`

2. **待機回数スコア (Coldness / Absence Score)**
   - 対象の番号が最後に当せんしてからの空き回数。
   - 長らく出ていない番号（コールドナンバー）は「そろそろ出やすい」とみなして少し優遇する。重み付け: `x1.5`

3. **相関性スコア (Correlation Score)**
   - **※1つ目以降の番号を選ぶ際にのみ計算**
   - 既に選出された番号リストと、過去に「同時に当せんした回数」が多い番号を評価する。
   - 重み付け: `x1.2`

## 抽選のフロー

1. 初期状態として、1〜37までの番号プールを用意します。
2. 以下の計算をループさせ、プールから番号が7つ選ばれるまで繰り返します。
   1. 残っているすべての番号に対して、上記の3指標をもとに**総合スコア**を計算します。
   2. 計算された各番号の総合スコアを比較し、**最もスコアが高い1つの番号**を選出（プールから除外）します。
   3. 次のループへ進みます。この際、「相関性スコア」が確定した番号に基づいて毎回再計算・更新されます。

## UI上の表示理由について

選出された7つの番号には、どの要素でスコアを大きく稼いで選ばれたかを示すラベルが付与されます。

- **長期待機**: 最後に当せんしてからの空き回数が相対的に長く、待機スコアが特出して高かった場合。
- **高頻度**: 過去の当せん回数が非常に多く、頻度スコアが特出して高かった場合。
- **相性よし**: 既に当せん番号として選出された直前の番号たちと過去の同時出現回数が多く、相関スコアで後押しされた場合。
- **連続出現中**: 直近の抽せんで2回以上連続で当せんしている番号で、勢いがあると判断された場合（スコアには直接関与しませんが、理由として最優先で表示）。
- **統計バランス**: 上記のどれにも極端に偏ることなく、無難なバランスで引き当てられた場合。

> 当アルゴリズムは乱数を使用せず、過去のデータに基づく完全なスコア順（相関性による動的スコア加算を含む）で選出するため、データが更新されない限り「予測する」ボタンで提案される番号は常に同じものになります。
